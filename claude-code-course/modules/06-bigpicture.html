常见故障排查

现在你已经能看到整张地图了，让我们来聊聊出问题时会发生什么。以下是用户最常遇到的五个问题——以及在架构中该到哪里去排查：

1
Claude 没有响应

检查 API 层（services/api/）。可能是认证问题、速率限制或网络故障。这属于区域 3——你的机器和云端 Claude 之间的连接。

2
Claude 有回复但没有做出更改

检查工具系统（services/tools/）。可能是权限阻止了执行。Claude 知道该做什么，但安全门控拦住了它——看看区域 4。

3
Claude 似乎忘记了之前的上下文

检查上下文压缩（services/compact/）。对话可能被自动摘要了，丢失了具体细节。这是因为上下文窗口空间不够了。

4
启动缓慢或崩溃

检查 init()（entrypoints/init.ts）。配置解析错误或代理问题是常见原因。这属于区域 5——其他一切都依赖的地基。

5
Claude 陷入循环，没有进展

检查查询循环（query.ts）。可能需要调整最大轮次限制或 token 预算。Claude 可能在重复同一个操作，或者对话状态卡在了一个死循环里。

💡
调试思维

注意到了吗？每个问题都能映射到架构中的一个特定区域。这就是理解全局的力量：当出了问题时，你不用慌张，而是可以冷静地问一句"这属于哪一层？"，然后把排查范围缩小到正确的"街区"。这正是专业开发者调试的方式。

值得了解的设计模式

在整个课程中，你反复看到了相同的工程模式。这些并非 Claude Code 独有——它们是整个软件行业久经考验的策略。了解它们，你就能读懂专业软件是怎么构建的。

🔁
Agent Loop（智能体循环）

AI 做出决策、执行动作、观察结果、然后重复。这是所有 AI 编程工具背后的核心模式，也是让智能体区别于简单聊天机器人的关键。

⚡
Parallel Prefetch（并行预取）

提前启动耗时操作，稍后再收集结果。用于启动优化——同时发起网络请求和文件读取，这样就没有任务需要排队等待。

🧠
Memoization（记忆化）

计算一次，永远复用。用于 init() 和许多内部缓存。既然答案没变，为什么要重新计算一遍呢？

✂️
Feature Flags（功能标志）

在编译时开关功能，彻底去除未使用的代码。不会发布任何死代码——如果功能门控关闭了，那段代码在构建中就不存在。

🔒
Permission Gates（权限门控）

操作之前，必先请求许可。这是 AI 工具的安全原则。每个破坏性操作都必须通过检查，让人类拥有最终决定权。

🗜️
Context Compaction（上下文压缩）

摘要旧对话以适应 AI 记忆限制。用细节换容量——就像把会议记录浓缩成要点，这样才能继续讨论。

📺
Terminal React（终端 React）

用 React（一个 Web UI 框架）在终端中渲染丰富的界面。组件化、状态管理、声明式渲染——只不过是在命令行里而不是浏览器中。

🔄
Async Generators（异步生成器）

逐段传输数据，而不是等到全部完成。用于对话循环——你能实时看到 Claude 思考时逐字出现的文字。
