什么是 Claude Code？

一台拥有超能力的对讲机——你对着它说话，一位天才助手回答你，而且它还能伸出手来帮你搞定事情。

终端里的 AI 助手

你打开终端，输入 claude，然后开始和一个能阅读你代码库、编辑文件、执行命令的 AI 聊天。但当你按下回车到看到回复之间，到底发生了什么？

Claude Code 是 Anthropic 打造的一款命令行 AI 编程助手。你用自然语言——就是日常说的大白话——提问或下达指令，它就能阅读你的代码、编辑文件、执行命令。就好比有一位资深开发者在和你结对编程，只不过他住在你的终端里。

💡
终端 vs. 浏览器

跟 ChatGPT（在浏览器中运行）不同，Claude Code 是一个 CLI 程序，直接在你的电脑上运行。这很关键，因为它可以直接访问你本地的文件和终端——再也不用在聊天窗口和编辑器之间来回复制粘贴代码了。

🧠
理解代码

读取你的文件，了解项目结构，理清各个模块之间的关系——就像一个刚把你整个代码库通读一遍的开发者。

✏️
编写代码

直接在你的电脑上编辑、创建和删除文件。再也不用从聊天窗口复制代码片段粘贴到编辑器里了。

⚡
执行命令

执行 Shell 命令、安装依赖、跑测试、检查结果——然后根据执行结果决定下一步该怎么做。

按下回车后发生了什么

当你输入一条提示词，比如 "修复 auth.ts 里的 bug"，然后按下回车，幕后会触发一整条事件链。让我们跟踪这条消息的完整旅程：

1
你输入提示词

你在终端里写下 "修复 auth.ts 里的 bug"，然后按下回车。

2
REPL 捕获你的输入

终端 UI（基于 React + Ink 构建）获取你的文本并准备进行处理。

3
上下文和你的消息打包在一起

你的提示词会被附加上额外的上下文：一个系统提示词（给 Claude 的指令）、过往对话的记忆，以及项目信息。

4
发送到 Claude 的 API

打包好的内容通过互联网发送到 Anthropic 的服务器，由 Claude AI 模型进行处理。

5
Claude 流式返回回复

回复一个 token 一个 token（逐词）地到达——而且可能包含使用工具的请求，比如编辑某个文件。

6
工具执行

如果 Claude 判断需要做点什么——比如读取文件或者应用一个编辑——工具系统就会在你的电脑上执行相应操作。

7
结果反馈回去

工具的执行结果（比如文件内容，或者"编辑已成功应用"）会被发送回 Claude，作为下一轮推理的输入。

8
最终回答

Claude 给出一个人类可读的回答："我在 auth.ts 第 42 行发现并修复了空值检查的 bug。"

✨
智能体循环（Agent Loop）

这种模式——AI 决定该做什么、执行操作、观察结果、然后再次决策的循环——被称为智能体循环。这是现代 AI 中最重要的概念之一。Claude Code 不只是在回答问题；它在一个持续的循环中进行推理、执行操作、不断调整。第 5–7 步可能会重复很多次，直到 Claude 给出最终答案。

技术栈

Claude Code 由一组精心挑选的技术构建而成。下面是驱动它运行的核心技术：

TS
TypeScript

所有代码所使用的编程语言——带类型安全的 JavaScript。

⚛️
React + Ink

在终端中渲染 UI——注意，不是浏览器！Ink 是一个库，让你可以用 React 组件来构建终端界面。

🥟
Bun

JavaScript 运行时和打包工具——类似 Node.js，但更快。负责运行和打包代码。

🤖
Anthropic SDK

与 Claude AI 大脑对话的官方库——负责发送提示词和接收回复。

⌨️
Commander.js

解析命令行参数——当你运行 claude 时，它能识别 --model 或 --verbose 这样的标志位。

replLauncher.tsx export async function launchRepl( root: Root, appProps: AppWrapperProps, replProps: REPLProps, renderAndRun: (root: Root, element: React.ReactNode) => Promise<void> ): Promise<void> { const { App } = await import('./components/App.js'); const { REPL } = await import('./screens/REPL.js'); await renderAndRun(root, <App {...appProps}> <REPL {...replProps} /> </App>); }
通俗解读
第 1 行：定义一个叫 launchRepl 的函数，用来启动交互式终端界面。它需要屏幕（root）、应用设置、REPL 设置，以及一个渲染方法。
第 2–7 行：加载两个核心组件——App 包裹层（提供全局状态）和 REPL 屏幕（也就是实际的聊天界面）。它们是按需懒加载的，这样启动速度更快。
第 8–10 行：把 REPL 屏幕嵌套在 App 包裹层里面进行渲染——就像嵌套 HTML 标签一样，只不过这里渲染在你的终端里而不是浏览器里。

这是 Claude Code 代码库中的真实代码。注意它使用 React 组件（<App>、<REPL>），跟开发 Web 应用一模一样——只不过所有内容都渲染在你的终端里，而不是浏览器中。
